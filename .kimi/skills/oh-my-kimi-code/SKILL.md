# Oh My Kimi - 代码分析

专注于 Python 代码结构分析和项目统计的工具集合。

## 工具概览

| 工具 | 用途 | 典型场景 |
|------|------|----------|
| `code_analyze` | Python 代码结构分析 | 分析文件/目录结构、统计函数类、计算复杂度 |
| `project_stats` | 项目统计 | 代码量统计、语言分布、目录结构分析 |

---

## code_analyze - Python 代码分析

使用 AST 分析 Python 代码结构，统计函数、类、导入和复杂度。

### 核心参数
- `path` (必需): 文件或目录路径
- `include_tests`: 是否包含测试文件，默认 false

### 典型用法
```json
// 分析单个文件
{"path": "./my_module.py"}

// 分析目录，包含测试文件
{"path": "./src", "include_tests": true}
```

### 输出内容
- **文件级别**: 总行数、代码行数、空行数、注释行数
- **函数列表**: 函数名、行号、参数数量、是否有文档字符串
- **类列表**: 类名、基类、方法数量、是否有文档字符串
- **导入语句**: 导入的模块和名称
- **复杂度评分**: 基于函数数、类数和分支复杂度计算

### 适用场景
- 了解代码库结构
- 识别复杂度过高的文件
- 统计代码规模
- 代码审查前的概览
- 检查代码文档覆盖率

## project_stats - 项目统计

统计项目代码量和结构，按语言、目录分组。

### 核心参数
- `path`: 项目目录，默认当前目录
- `exclude`: 排除模式，默认 `"node_modules,__pycache__,.git,venv,.venv,dist,build"`

### 典型用法
```json
// 分析当前目录
{"path": "."}

// 自定义排除模式
{"path": ".", "exclude": "node_modules,vendor,*.min.js"}
```

### 输出内容
- **按语言统计**: 文件数、代码行数（Python, JavaScript, TypeScript 等）
- **按目录统计**: 文件分布情况
- **最大文件列表**: 按行数排序的文件
- **总计**: 总行数、代码行数、空行数

### 适用场景
- 了解项目技术栈
- 评估项目规模
- 识别代码分布
- 查找大文件

## 组合使用技巧

### 代码审查流程
```
1. project_stats "." → 了解项目整体情况
2. code_analyze "./src" → 分析核心代码结构
3. 重点关注：
   - 复杂度评分高的文件
   - 缺少文档字符串的函数/类
   - 代码行数过多的文件
```

### 项目健康检查
```
1. project_stats → 查看语言分布
2. code_analyze → 检查：
   - 平均复杂度
   - 文档覆盖率
   - 函数/类数量
```

## 常见任务速查

| 任务 | 工具 | 参数示例 |
|------|------|----------|
| 分析单个文件 | code_analyze | `{"path": "main.py"}` |
| 分析整个项目 | code_analyze | `{"path": ".", "include_tests": true}` |
| 统计代码行数 | project_stats | `{"path": "."}` |
| 查看技术栈 | project_stats | `{"path": "."}` |
| 找大文件 | project_stats | 查看 "largest_files" |
| 检查复杂度 | code_analyze | `{"path": "./src"}` |

---

## 注意事项

1. **code_analyze** 只支持 Python 文件（.py）
2. **project_stats** 支持所有文本文件，自动识别语言
3. 工具自动排除常见目录（node_modules, __pycache__, .git 等）
4. 大型项目分析可能需要一些时间
